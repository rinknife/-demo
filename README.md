# AI Shopping Agent - 智能购物决策助手
基于**大模型+Function Calling+多轮对话记忆**的电商场景AI智能体，可自动理解用户购物意图、精准筛选商品、专业分析对比，为用户提供高参考性的购物决策依据，实现类真人导购的自然交互体验。

## 项目定位
面向电商场景的AI智能体（Agent）实战Demo，核心解决用户购物时**需求表达模糊、商品筛选繁琐、对比决策困难**的痛点，具备**意图自动识别、工具智能调用、上下文记忆、结构化推荐**的核心能力，是大模型Function Calling落地电商场景的典型实践。

## 一、旧版核心问题
旧版为基础功能实现版本，存在**意图识别粗糙、对话体验生硬、功能联动弱、鲁棒性低**等问题，具体表现为：
1. **意图识别能力薄弱**：仅支持简单关键词匹配，无法覆盖多样化的用户表达（如“1500元以下”“不超过2000”无法识别为价格上限，“笔记本”无法映射为“电脑”品类）；
2. **对话记忆缺失**：无多轮对话上下文继承能力，用户追问“哪个性价比高”时，需重新输入需求，无法基于上一轮推荐商品分析；
3. **功能调用单一**：仅支持单一步骤的函数调用，无法实现“价格+品牌+品类”的组合筛选，且无价格反馈的动态调整能力；
4. **交互体验被动**：仅机械响应用户需求，无主动引导、无异常处理（如无符合条件商品时仅简单提示，无后续建议）；
5. **商品处理简单**：无商品打分排序逻辑，推荐结果无优先级，且未考虑库存、评分等实际购物影响因素；
6. **测试体系缺失**：无标准化的测试用例和效果评估机制，无法量化验证智能体的回复准确率和实用性。

## 二、新版核心更新
新版针对旧版问题进行**全链路重构与能力升级**，重点强化**意图识别、上下文记忆、函数联动、体验优化、测试体系**五大核心模块，实现从“基础功能实现”到“实用型智能体”的跨越，具体更新如下：

### 2.1 精准意图识别：覆盖多样化用户表达
重构`extract_user_intent`核心函数，支持**价格、品牌、品类、对比、价格预警**五大类意图的结构化提取，解决旧版表达覆盖不足问题：
- 价格提取：支持“X元以内/以下/不超过/预算X”等多句式，新增“价格宽松匹配”标记（如“X元左右”自动放宽10%筛选范围）；
- 品牌映射：实现别名自动转换（如“苹果”→“Apple”），支持从商品名称中提取品牌（如“iPhone15”→“Apple”）；
- 品类归一：支持品类同义词映射（如“笔记本”→“电脑”、“蓝牙耳机”→“耳机”）；
- 对比/价格预警：优化正则匹配逻辑，支持“对比A和B”“A和B哪个好”“X值不值得买”“X价格合适吗”等常见表达，精准提取目标商品。

### 2.2 多轮对话记忆：实现上下文状态继承
新增**对话状态持久化**能力，基于聊天历史自动继承上一轮的**品类、品牌**筛选条件，支持**价格反馈动态调整**，让对话更自然：
- 状态继承：若用户未指定新的品类/品牌，自动沿用上一轮推荐商品的品类/品牌（如先问“小米手机”，再问“2000元以内”，自动筛选“2000元以内的小米手机”）；
- 价格反馈：用户说“太贵”“超出预算”时，自动以上一轮推荐商品的最低价为基准，下调10%作为新的价格上限并重新筛选；
- 犹豫复用：用户表达犹豫时（如“不知道选哪个”“帮我选一个”），直接复用上一轮推荐商品，无需重新调用工具筛选，提升交互效率。

### 2.3 智能函数联动：支持组合条件筛选与多步骤调用
重构函数调用逻辑，从“单一函数调用”升级为**“组合条件链式筛选”+“多步骤函数调用”**，适配复杂购物需求：
- 组合筛选：支持“价格+品牌”“价格+品类”“品牌+品类”“价格+品牌+品类”多维度组合筛选，按“价格→品牌→品类”优先级逐步缩窄商品范围；
- 自动兜底：某一维度筛选无结果时，自动触发宽松策略（如价格筛选无结果时，放宽10%价格上限重新筛选）；
- 多步骤调用：新增`agent_loop`函数，支持最多3轮的自动函数调用，若首轮筛选结果不足，自动补充调用其他工具完善数据。

### 2.4 商品智能排序：基于多维度打分的精准推荐
新增`score_products`商品打分排序函数，从**匹配度、库存、评分**三个维度为商品打分，推荐结果按优先级排序，告别无差别推荐：
1. 价格匹配度（3分）：符合价格上限得满分，宽松匹配得1分；
2. 品牌匹配度（2分）：符合品牌偏好得满分；
3. 基础品质（2分）：库存大于0得1分，评分≥4.5得1分；
- 最终按得分从高到低排序，取前5个商品作为推荐结果，确保推荐的商品**匹配需求、品质可靠、库存充足**。

### 2.5 交互体验优化：从“被动响应”到“智能引导”
优化系统提示词和回复逻辑，新增**异常友好提示、结构化推荐、专业决策分析**，提升交互体验：
- 异常处理：无符合条件商品时，根据用户筛选条件给出具体建议（如“2000元以内无小米手机，建议提高预算或尝试红米系列”）；
- 结构化输出：推荐商品附带**价格、品牌、库存、店铺**核心信息，推荐理由简洁专业，贴合真人导购的表达逻辑；
- 专属场景处理：对比需求直接输出结构化参数对比表，价格预警需求直接给出“值得买/观望”的明确结论+历史价格依据。

### 2.6 工程架构重构：模块化解耦，易扩展易维护
对项目架构进行**全模块化拆分**，将核心能力解耦为独立文件，解决旧版代码耦合度高、难以扩展的问题，架构如下：
```
├── .env                # 环境配置文件（阿里云百炼API Key）
├── app.py              # 主程序：Gradio交互界面+对话入口
├── agent.py            # 核心智能体：对话流程控制+意图处理+推荐逻辑
├── config.py           # 全局配置：API地址、模型参数、超参数
├── functions.py        # 工具函数：Function Calling元数据+函数执行逻辑
├── prompts.py          # 提示词管理：系统提示词、函数决策提示词统一维护
├── shop_data.py        # 数据层：模拟商品库、店铺库、辅助数据函数
├── utils.py            # 工具层：意图提取、商品摘要、打分排序等通用函数
├── test.py             # 测试体系：自动化测试用例+大模型评测+结果统计
└── requirements.txt    # 项目依赖包
```

### 2.7 标准化测试体系：量化验证智能体效果
新增`test.py`自动化测试模块，实现**测试用例自动生成、大模型客观评测、结果统计分析**，可量化验证智能体的回复准确率和实用性：
- 自动生成测试用例：基于模板随机生成覆盖**价格约束、品牌约束、品类约束、对比、价格预警、模糊意图**等6大场景的测试用例；
- 大模型裁判评测：调用大模型作为“评测专家”，从**需求理解、商品匹配、回答质量**三个维度客观评判回复是否合格；
- 结果统计分析：自动计算**通过率**，统计**失败原因TOP榜**，输出失败案例详情，为后续优化提供明确方向。

### 2.8 核心功能增强：覆盖更多电商购物场景
在旧版**品类搜索、价格筛选、品牌筛选**的基础上，新增/优化核心功能：
- 商品参数对比：支持两款商品的核心参数（价格、处理器、屏幕、摄像头、电池）结构化对比，给出直观选购建议；
- 价格预警分析：基于商品30日均价、历史最低价，判断当前价格是否“值得买”，支持用户心理价位自定义；
- 店铺信息查询：可调用工具获取店铺评分、是否官方、售后服务等信息，为购物决策提供店铺维度依据；
- 商品摘要生成：自动将筛选后的商品转换为大模型易理解的摘要格式，提升推荐准确性。

## 四、快速运行
### 4.1 环境准备
1. 安装Python 3.8+环境；
2. 安装项目依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 配置阿里云百炼API Key：在`.env`文件中填写`BAILIAN_API_KEY`（从阿里云百炼平台获取）。

### 4.2 启动项目
```bash
python app.py
```
4. 浏览器访问`http://localhost:7860`，即可进入交互界面开始使用。

## 六、当前版本仍存在的缺点
新版虽实现了核心能力的升级，但仍为**电商场景Demo级实现**，存在以下待优化的问题，也是后续核心改进方向：
1. **意图识别仍有局限**
   - 不支持多品牌同时筛选（如“小米和华为手机”仅能识别第一个品牌）；
   - 无法识别用户的**使用场景需求**（如“游戏手机”“拍照手机”“备用机”）；
   - 正则匹配仍存在边缘案例覆盖不足（如“1k5左右”“两千出头”等口语化价格表达）。

2. **对话记忆能力有限**
   - 仅能继承上一轮的**品类、品牌**，无法记忆用户的**长期偏好**（如“喜欢轻薄机身”“需要大电池”）；
   - 无对话历史的主动总结能力，多轮追问后易丢失早期需求。

3. **商品库与数据维度不足**
   - 商品品类仍较单一（仅手机、耳机，缺少电脑、平板、家电等品类）；
   - 商品数据维度缺失**真实电商核心信息**：月销量、好评率、差评关键词、促销活动、物流方式等；
   - 店铺库信息简单，无**店铺类型区分**（官方旗舰店/品牌店/第三方店）、售后服务差异等。

4. **智能体决策能力待提升**
   - 商品打分维度较简单，未考虑**配置性价比、用户使用场景、商品口碑**等核心因素；
   - 无**主动追问能力**，当用户需求模糊时（如“想买手机”），仅推荐热门商品，未主动询问预算/品牌/使用场景；
   - 函数调用为**固定优先级链式调用**，无动态规划能力（如根据用户需求自动调整筛选顺序）。

5. **交互体验仍有优化空间**
   - 推荐结果为纯文本形式，无**结构化表格/可视化对比**，信息展示不够直观；
   - 无**避坑提醒**，未对低评分、少库存、非官方店的商品进行风险标注；
   - 无**主动引导式交互**，回答结束后未提供可继续追问的方向（如“需要我帮你对比这两款的详细参数吗？”）。

6. **工程化与鲁棒性待加强**
   - 无**异常捕获的全链路覆盖**，部分边缘场景（如商品ID映射失败、函数调用返回空数据）会导致程序报错；
   - 无**配置化管理**，品类、品牌、打分权重等核心参数需在代码中修改，不易于非开发人员使用；
   - 未考虑**大模型调用失败/超时**的兜底策略，依赖大模型的稳定性。

7. **测试体系待完善**
   - 测试用例为模板生成，缺少**真实用户的对话语料**，测试结果与实际使用场景存在偏差；
   - 大模型评测的**评判标准较模糊**，无量化的评分体系（仅“通过/不通过”）；
   - 无**性能测试**，未验证大模型调用次数、响应时间等工程性能指标。

## 七、后续优化方向
基于当前版本的缺点，后续将从**意图理解、记忆能力、决策能力、体验优化、工程化、数据维度**六个方向进行深度升级，逐步将Demo升级为**可落地的实用型AI购物助手**：
1. **增强意图理解**：加入场景意图识别、多品牌同时筛选、口语化表达适配，引入语义理解模型替代纯正则匹配；
2. **升级记忆能力**：实现用户**长期偏好记忆**（预算、品牌、使用场景、配置偏好），新增对话历史总结能力；
3. **提升决策能力**：丰富商品打分维度（加入场景匹配度、口碑分），实现**主动追问**和**函数调用动态规划**；
4. **优化交互体验**：新增结构化表格/可视化对比、商品避坑提醒、主动引导式交互，支持个性化推荐；
5. **完善工程化**：实现核心参数配置化、全链路异常捕获、大模型调用兜底策略，提升系统鲁棒性；
6. **丰富数据维度**：扩充商品品类、补充真实电商数据（销量、好评率、促销等）、完善店铺信息与售后服务维度；
7. **深化测试体系**：引入真实用户语料、建立量化评分体系、增加性能测试，实现智能体效果的持续迭代优化；
8. **对接真实平台**：适配京东/抖音商城开放平台API，实现**实时商品、实时价格、实时评价**的获取，从Demo走向实际应用。
